<original_problem_statement>
The user, Aman Agarwal, wants to build a production-grade, private mobile app for his jewelry business, Yash Trade, for his network of ~40,000 wholesale and retail jewelers. The primary goal is to create a daily-use business utility that increases customer retention and repeat business.

**PRODUCT REQUIREMENTS:**

*   **Core App Experience:**
    *   An Instagram-style feed of jewelry photos/videos.
    *   A daily rates module for Gold and Silver.
    *   A robust rewards/loyalty system.
    *   Request Call and Request Video Call functionality.
    *   A Silver Jewellery Calculator.
    *   An AI-powered assistant.
    *   A customer portal for account details, order status, and rewards.
    *   Stories/highlights for daily updates.

*   **Architecture & User Roles:**
    *   A customer-facing mobile app (Expo React Native) for jewelers.
    *   A separate, web-based internal panel for all backend staff.
    *   **Admin:** Full control via the web panel.
    *   **Executive:** A limited-access panel for managing customer requests (calls, inquiries, cart submissions).
    *   **Billing Executive:** A highly restricted panel for manually managing customer reward points.
    *   **Jeweller (User):** Private access to the mobile app via OTP-verified phone numbers.

*   **Key Features & User Requests:**
    *   **Production-Grade Uploads:** Support for direct, bulk file uploads (10,000+ images) with batch management, progress bars, and scalable object storage.
    *   **Business Cart Workflow:** An Add to Cart feature that is a selection/shortlist tool. The submitted cart goes to the executive team for manual billing, not an e-commerce checkout.
    *   **Advanced Request Flow:** When making a request (e.g., video call), users can select multiple products from the catalog, add notes, and see the specific product information in the executive panel.
    *   **Manual Rewards System:** A dedicated Billing Executive role to manually add/deduct reward points for customers with a full audit trail.
    *   **Improved UI/UX:** Clickable stories, high-definition images in the feed, proper nested back-navigation, and clear success confirmations after user actions.
</original_problem_statement>
